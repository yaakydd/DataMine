Data Cleaning Logic

## IMPLEMENTATIONS

Task 1:
1. Correct data inconsistencies
- Display all the column names and print it together with the datatypes 
- Ask if the user wants to change any column names and dataypes
    .Allow user to change it and redisplay the column names
## All the column names must be in the same format
    .Fix spaces, symbols ($, %, !), and casing automatically

2. Handling Missing Data
- Print all the columns with missing data
    . Show the percentage of the missing data in each column and explain to the user what it means and suggest what they can do and what it means if 
    they do and what will happen if they don't
- Apply all the changes the user makes to handle the missing data
    . Make sure to bare in mind the type of topic and its relevance and what it means if they handle the missing data

3. Handling Duplicates
- Print the number of duplicated rows
    .Show the percentage of the duplicated rows in each row and explain to the user what it means and suggest what they can do and what it means if 
    they do and what will happen if they don't 
- Apply all the changes the user makes to handle the duplicated rows
    . Make sure to bare in mind the type of topic and its relevance and what it means if they handle the duplicated rows

4. Outlier Detection
- Use z-score or interquartile range to identify outliers
    . Explain to the user what outliers are
    . Depending on their topic, explain to them what these outliers identified(if any found)
    will do it their data. Suggest some fixes and also explain to them what their choices will do to the data if they choose any option from your options

5. Handling Skewed Data (Data Transformation)
- Apply a Log Transformation or Square Root Transformation to pull the outliers closer to the center and create a more "Normal" (Gaussian) distribution.
- Apply other transformation techniques relevant to their topic.

5. Validation and Cross-Checking
- This is the "sanity check" phase.
- Logic Checks: Does a "Year of Birth" column say 2026 for someone listed as an "Adult"?Range Constraints: Ensuring a "Percentage" column doesn't have a value of $110\%$.
