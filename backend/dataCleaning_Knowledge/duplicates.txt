Here's a full walkthrough of every decision.
df.duplicated(keep="first") returns a boolean Series where True marks every row that is an exact copy of a row that appeared earlier. The original first occurrence is always False. This is exactly what you want for counting — you're counting the extras, not the originals. duplicate_mask.sum() then counts how many True values there are.
The subset report is the proactive xAI feature. Instead of waiting for the user to notice that their email column has duplicate values, the code automatically scans every column whose name contains keywords like "id", "email", "phone" — columns that should logically be unique. If it finds repeats, it flags them with an explanation. This is what makes the tool feel intelligent rather than passive.
safe_json() is called on the sample rows before returning them. df.to_dict(orient="records") can still contain numpy types inside the values — things like np.int64(42) instead of plain 42. Without safe_json(), FastAPI would crash trying to serialise those.
drop_duplicates(keep=False) is different from keep="first" in an important way. With keep="first", if rows A, B, C are all identical, you keep A and remove B and C — you lose 2 rows. With keep=False, all three are removed — you lose 3 rows. The explainer makes this distinction explicit so the user understands what they are choosing.
reset_index(drop=True) is called after every drop operation. When pandas drops rows it leaves gaps in the index — after dropping row 5, your index jumps from 4 to 6. reset_index(drop=True) renumbers everything from 0 cleanly. Without it, row-based operations later can behave unexpectedly.
The subset parameter lets the user target specific columns instead of the full row. This matters when, say, an email column has duplicates but other fields like timestamp or notes differ — a full-row check would miss it, but a subset check on ["email"] catches it.
